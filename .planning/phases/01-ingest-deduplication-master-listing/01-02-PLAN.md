  <action>
    Create `utils/normalizers.js` and implement:
    1. `normalizeCompanyName(name)`: Converts to lowercase, strips corporate suffixes (like "Limited", "Ltd", "Corp", "Co", "IPO", "Details"), removes non-alphanumeric characters except spaces, and squashes multiple spaces into a single space.
    2. `normalizeSymbol(symbol)`: Converts to uppercase and strips exchange suffixes like `-BE`, `-EQ`, `.NS`, `.BO`.
    3. `parseIndianDate(dateStr)`: Standardizes various date strings (e.g. Indian exchange formats like `05-Jun-2026` or ISO formats `2026-06-05`) into native JavaScript Date objects.
    4. `formatDateISO(dateObj)`: Converts a Date object to a `YYYY-MM-DD` string.
    Export these functions using CommonJS `module.exports`.
    Create `test/normalizers.test.js` to unit test these functions with all edge cases (suffix stripping, different date formats) and verify they work.
  </action>

<threat_model>
### Trust Boundaries
- Network boundary: The connection between the local scraping process and external APIs (Upstox, NSE).
- File system boundary: The cookie storage file (`nse_cookies_http1.json`) on disk.

### STRIDE Threat Register
| Threat ID | Category | Description | Mitigation/Acceptance |
|---|---|---|---|
| T-01-SC | Spoofing / Tampering | Malicious third-party dependencies or upstream package poisoning. | Mitigate: Minimal third-party footprint; dependencies locked in `package-lock.json`. |
| T-01-02 | Spoofing / Tampering | Session cookies stored in `nse_cookies_http1.json` are tampered with or modified. | Mitigate: The file is written locally and readable only by the running user process. Input validations are performed when cookies are loaded. |
| T-01-03 | Denial of Service | External APIs (Upstox/NSE) rate-limit the pipeline due to high frequency requests. | Mitigate: Incorporate request delays (150ms sleep) and check existing records before requesting detailed data. |
</threat_model>
