  <files>run_pipeline.js, test/pipeline.test.js</files>
  <read_first>01-RESEARCH.md</read_first>
  <action>
    Write `run_pipeline.js` as the central ingestion runner. It must:
    1. Parse CLI arguments using `process.argv`: default to current calendar year (e.g. 2026), support `--year YYYY` and date ranges `--from YYYY-MM-DD --to YYYY-MM-DD`.
    2. Pull raw lists and detail payloads from Upstox and NSE sources.
    3. Standardize schemas to top-level fields: `isin`, `symbol`, `companyName`, `status`, `biddingStartDate`, `priceBand`, `documentUrls`, and raw sources nested under `raw_sources.upstox` and `raw_sources.nse`.
    4. Deduplicate hierarchical matches:
       - Match 1: ISIN matches exactly.
       - Match 2: Normalized Symbol matches exactly.
       - Match 3: Normalized Company Name has Jaro-Winkler score > 0.90 AND their bidding start dates are within 30 days of each other.
       - Log borderline/review-flagged matches separately.
    5. Merge properties using precedence: NSE-BSE SDK dates/documents take precedence, while Upstox numeric price band, size, lot sizes, allocation data takes precedence.
    6. Persist results atomically to `ipo_master.json`.
    7. Write tests in `test/pipeline.test.js` validating date filtering, matching rules, precedence rules, and schema structures.
  </action>
